tamagotchi

aka a reignited interest!!

Having recently re-discovered tamagotchis thanks to other personal websites, I got the new one for myself. With the excitement of my new nifty virtual pet, I wanted to make a little shrine dedicated to the tamagotchis I own and have raised!

version 1.0 › september 2024 — ???

This is probably one of my most ambitious layouts to date. I knew I wanted something very tactile-looking and real, incorporating images and stickers in a collage-like fashion to mimic a diary of sort. After all, I had my first tamagotchi when I was a kid, around the age I still kept a physical diary. Although I love making collages, I just felt like collage-based websites are often difficult to make responsive, which is why I tended to avoid them and stuck to more modern and minimal web designs previously. However, I spent some time thinking about the layout before executing it. Although it may not be the best solution, I think it still works!

The code for having the centered notebook with a few different divs overlaid on top of it took a bit of tinkering, but primarily used the same code base as my map of the internet page to have responsive divs on top of an image. This made use of this solution on stackoverflow and a responsive image map generator. The rotated divs I used for the side coloured tabs navigation required some trial and error to get into place because the rotation changes the position slightly, but I managed somehow.

Because the notebook is wider than it is tall, I knew this would pose a problem translating into a mobile layout. I've seen some solutions where they would rotate the entire notebook sideways, so a user would have to view the website sideways on their phone, but I didn't want to do that. Instead, I opted to crop the image to only reveal half of the open ntebook while on mobile. This way, it would be taller than it is wide instead, making it a bit closer to a mobile screen. This is actually the reason why I only used the right page on the desktop design for text: I didn't want to have to fiddle with two content divs when I had to make it responsive, so instead I utilized the left page by designing a collage that could be easily cropped for mobile view without having to change too much of the actual code structure for the content of the page to make it responsive.

Although there are aspects which I think I could've done better, I'm quite proud of this page! The next thing I'd like to do is probably change out the link colours from the default ones. I also have plans to better divide the pages as I add more content, since the actual space for content is the size of the paper in the image, which is quite small and would require a lot of scrolling once I add more things.
